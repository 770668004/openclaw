# 技能进化管理系统 (Skill Evolution Manager)

## 概述

这是一个智能的技能管理系统，能够自动对现有技能进行分类、审计、整合和升级。系统按照24小时执行周期运行，但优先级始终低于当前正在执行的任务。

## 功能特性

### 1. 智能分类
- 自动将63个现有技能分为8个功能类别
- 基于技能的实际用途而非技术实现进行分类
- 支持动态调整分类规则

### 2. 严格审计
- 升级前进行全面的代码审计
- 安全性检查和漏洞扫描
- 逻辑一致性验证

### 3. 智能进化
- 基于新学习的知识进行优化
- 自动识别重复功能并进行整合
- 性能优化和安全增强

### 4. 24小时执行周期
- 定期自我升级
- 任务优先级管理（当前任务 > 进化任务）
- 自动回滚机制

## 功能分类

系统将技能分为以下8个类别：

1. **安全与审计** - 系统安全、风险评估、健康检查
2. **文件管理** - 备份、恢复、文件操作
3. **开发工具** - GitHub操作、代码审查、版本控制
4. **智能代理** - 编码代理、AI助手、自动化任务
5. **通信协作** - 消息发送、通知、群组管理
6. **数据获取** - 网络搜索、天气信息、API调用
7. **系统管理** - 进程管理、节点控制、资源监控
8. **实用工具** - 技能发现、格式优化、通用功能

## 使用方法

### 手动触发进化
```bash
# 执行完整的技能进化流程
python -m skill_evolution_manager --mode full

# 仅执行分类和审计
python -m skill_evolution_manager --mode audit

# 仅执行升级
python -m skill_evolution_manager --mode upgrade
```

### 自动调度
系统会自动在24小时周期内执行进化任务，无需手动干预。

### 查看状态
```bash
# 查看当前技能分类
python -m skill_evolution_manager --status

# 查看进化历史
python -m skill_evolution_manager --history
```

## 安全保证

1. **审计先行** - 所有升级操作前必须通过完整的审计流程
2. **备份机制** - 升级前自动备份原始技能文件
3. **回滚能力** - 升级失败时自动回滚到之前版本
4. **权限控制** - 需要用户明确授权才能执行破坏性操作
5. **沙箱测试** - 新技能在隔离环境中测试后再部署

## 配置选项

配置文件 `config.json` 支持以下选项：

- `execution_cycle_hours`: 执行周期（默认24小时）
- `task_priority`: 任务优先级设置
- `auto_backup`: 是否自动备份（默认true）
- `audit_level`: 审计级别（basic/standard/full）
- `categories`: 自定义分类规则
