# 智能技能匹配示例

## 基础命令匹配示例

### 文件操作类
- **用户输入**: "帮我备份这个文件夹"
- **匹配技能**: backup
- **理由**: 包含"备份"关键词，明确的文件操作需求

- **用户输入**: "整理一下我的文档"
- **匹配技能**: file-maintenance  
- **理由**: "整理"暗示文件组织和维护操作

### GitHub相关
- **用户输入**: "查看这个PR的状态"
- **匹配技能**: github-upgraded
- **理由**: 提及PR (Pull Request)，GitHub特定操作

- **用户输入**: "修复这个GitHub issue"
- **匹配技能**: gh-issues
- **理由**: 明确提到GitHub issue修复

### 系统管理
- **用户输入**: "检查我的系统安全"

- **理由**: "安全检查"是healthcheck的核心功能

- **用户输入**: "优化我的服务器性能"
- **匹配技能**: healthcheck-upgraded
- **理由**: 性能优化属于系统健康检查范畴

### 天气查询
- **用户输入**: "今天天气怎么样？"
- **匹配技能**: weather
- **理由**: 直接的天气查询

- **用户输入**: "明天需要带伞吗？"
- **匹配技能**: weather  
- **理由**: 隐含的天气预报需求

### 技能管理
- **用户输入**: "创建一个新的技能"
- **匹配技能**: skill-creator
- **理由**: 明确的技能创建需求

## 复杂场景匹配示例

### 多技能组合场景
- **用户输入**: "帮我分析这个代码库的问题并创建修复PR"
- **匹配技能**: [gh-issues, github-upgraded]
- **理由**: 需要先分析问题(gh-issues)，然后创建PR(github-upgraded)

- **用户输入**: "备份我的项目然后检查安全性"
- **匹配技能**: [backup, healthcheck-upgraded]
- **理由**: 两个独立但相关的操作，需要按顺序执行

### 上下文感知匹配
- **用户输入**: "播放音乐" (在Sonos环境中)
- **匹配技能**: sonoscli
- **理由**: 结合环境上下文，音乐播放指向Sonos控制

- **用户输入**: "播放音乐" (无特定环境)
- **匹配技能**: None (使用基础功能)
- **理由**: 缺乏具体上下文，无法确定特定技能

### 模糊意图解析
- **用户输入**: "让我的工作更高效"
- **匹配技能**: skill-evolution-manager
- **理由**: 效率提升通常涉及技能优化和自动化

- **用户输入**: "我需要更好的组织方式"
- **匹配技能**: file-maintenance
- **理由**: 组织需求通常指向文件和信息管理

## 特殊情况处理

### 冲突解决
当多个技能都可能匹配时：
1. 选择最具体的技能
2. 考虑用户历史偏好
3. 优先选择最近使用过的相关技能
4. 如仍无法确定，询问用户澄清

### 无匹配情况
当没有合适的技能匹配时：
1. 使用基础AI能力直接响应
2. 建议可能相关的技能
3. 询问是否需要创建新技能

### 新技能发现
当检测到重复的新需求模式时：
1. 记录需求模式
2. 建议创建新技能
3. 提供技能创建模板
